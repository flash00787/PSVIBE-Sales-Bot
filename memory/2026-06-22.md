## Key Learnings
1. **DB times are MMT** (UTC+6:30), not UTC
2. **`now_mmt()` usage**: API stores session start times as MMT
3. **Bot timer calc**: `_start.replace(tzinfo=MMT)` treats stored MMT correctly
4. **Cloudflare CDN caching**: Even with origin headers, CDN caches static assets
5. **Vite hash stability**: Same source code → same chunk hashes → stale CDN cache
6. **BotState collisions**: IntEnum auto-assigns values → must manually verify uniqueness
7. **Timeline time-only strings**: `/api/bookings` returns `startTime`/`endTime` as `"HH:MM"` — never use `new Date()` on these
8. **Python `bool(0) == False`**: `x or default` pattern breaks on zero values → use `x if x is not None`
9. **`%%` pattern doesn't work with `mysql.connector`**: uses `%s` parameter style

### 25. New Bug Pattern — Empty Set Fallthrough
**Pattern:** Functions returning empty collections (`[]`, `{}`) instead of raising exceptions:
- `get_consoles_from_setting()` → API fail → `[]`
- `fetch_console_status()` → API fail → `[]`
- Set comprehension on `[]` → `set()` (not exception)
- `set() or fallback` → `fallback` (Python `bool(set()) == False`) — this one works!
- But validation: `if cid not in valid` where `valid = set()` — ALL inputs rejected

**Prevention:** Never rely on `try/except` alone when a function returns empty collections on failure. Always check `if not result:` after the call.
