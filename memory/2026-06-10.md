#### 4. 🟡 Cron Job Adjustments — Reduce Session File Lock Conflicts
- `lock-monitor-cleanup`: 30s → **5 min** (300s) — was running every 30s, causing `EmbeddedAttemptSessionTakeoverError`
- `session-cleanup`: 5 min → **10 min** (600s) — reduce contention

### Root Cause Summary
- HEARTBEAT.md/AGENTS.md "stay quiet" rule was misinterpreted to apply to **incoming message responses**, not just heartbeat outreach
- lock-monitor every 30s created session file write conflicts with main session
- Gateway auto-restart (03:19 MMT) caused brief unavailability during night hours
- Session file count at 1,305 files / 446MB (close to 500MB limit)

### Lessons Learned
1. NEVER let "stay quiet" rules affect message responses — only heartbeat/outreach
2. Session file lock cron jobs (30s interval) directly conflict with main session → must be >60s
3. Always explicitly document: "Boss messages = ALWAYS respond" in golden rules
