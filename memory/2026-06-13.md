### PNL Redesign — Game Profit / Food Profit Split
**Before (wrong):**
- `food_revenue = gross - amount` → ❌ console multiplier counted as food
- `game_revenue = net - food_amt` → ❌ game underreported

**After (correct):**
- **Food Revenue** = `stock_out` table (actual items sold × unit_price)
- **Game Revenue** = `sales_daily.net` - Food Revenue (food already in sales total)
- **Food Profit** = Food Revenue - Food COGS (from FIFO)
- **Game Profit** = Game Revenue (no direct COGS)
- `gross_profit` kept as total number for dashboard backward compat
- New fields: `game_profit`, `food_profit` added as top-level fields

## Key Events

### 🔴 Bug 1: Kora Dashboard Login Not Working (JS Syntax Error)
- **Root cause:** `showUserActivity()` function in `<script>` block had broken quoting: `onclick="showUserActivity('' + u.tg_id + '','' + name.replace(/'/g, '\\'') + '')"` — the `''` inside single-quoted onclick caused a JS parse error
- **Impact:** ENTIRE `<script>` block failed to parse → login button `onclick="doLogin()"` never bound → init IIFE never ran → login impossible
- **Fix:** Rewrote as proper escaping: `onclick="showUserActivity(\'' + escapeHtml(String(u.tg_id)) + '\',\'' + escapeHtml(name.replace(/'/g, "")) + '\')"`
- **Lesson:** A JS parse error in any part of a `<script>` block kills ALL code in that block. Even unrelated functions like `doLogin()` stop working.

### 🔴 Bug 2: Kora Dashboard Members Empty
- **Root cause:** `fetch_members` API endpoint (`/api/fetch_members`) in `app.py` required API key (X-API-Key header) but Kora Dashboard sends JWT Bearer token
- **Fix:** Modified `verify_api_key()` in `app.py:169` to fallback to JWT Bearer token decode when X-API-Key header is missing. Now accepts both auth methods.
- **Also:** Changed endpoint to return rich objects (name, phone, tier, liability_mins, wallet_balance) instead of just member ID strings

### 🔴 Bug 3: Web Finance Members Not Showing
