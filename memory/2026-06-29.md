### Lessons Learned
- **#67: FormData.get() inside forEach is wasteful** — create FormData once, use `fd.get()` within the callback
- **#68: PATCH body always includes all Optional fields** — Pydantic Optional[int] defaults to None. If edit form only sends one field, the other gets cleared to NULL. Must always send BOTH from_account and to_account from the record's existing values
- **#69: `from_account_id` was no-op conditional bug** — `X if True else X` in tx_create always passed through. Must use `X if condition else None` pattern
- **#70: Payment creation must respect tx_type** — Buy→money FROM us (from_account_id), Sell→money TO us (to_account_id). All 4 payment paths (tx_create, dashboard modal, dashboard R/P, bot) had this bug
- **#71: Opening balance SET overrides accumulated balance** — Using `PATCH /api/accounts/{id}/balance` replaces the balance entirely. If payments were created BEFORE the balance was set, their balance effects are overridden
