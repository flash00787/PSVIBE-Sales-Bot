## üêõ Fix: Morning Health Summary MySQL Query Bug

**Root cause:** `lib/ssh_vps.js` ‚Üí `mysqlQuery()` used `2>&1` to merge stderr into stdout. MySQL password warning (`"mysql: [Warning] Using a password on the command line interface can be insecure."`) became the first line of the output. `auto_maintenance.js`'s `morningHealthCheck()` parsed `lines[0]` as column headers (got the warning instead) and `lines[1]` as data (got the actual header). Result: all values parsed as garbage ‚Üí showed placeholder "0" values.

**Fix:** Made `mysqlQuery()` `async` and added a filter:
```
result.replace(/^.*Warning.*\n?/m, '')
```
This strips any "Warning" line from the output before returning.

**Verification:** Tested with `test_mysqlQuery.js`:
- `SELECT COUNT(*) as txns FROM sales_daily WHERE sale_date = '2026-06-14'` ‚Üí `43` ‚úÖ
- Full multi-column query: `txns=43, total=737950.00, gross=789750.00, discounts=51800.00` ‚úÖ
- Morning report will show correct data at next 8 AM MMT run

**File modified:** `/home/node/.openclaw/workspace/lib/ssh_vps.js` (lines 49-55)
- Changed from: `function mysqlQuery(query) { return sshExec(...2>&1...); }`
- Changed to: `async function mysqlQuery(query) { const r = await sshExec(...2>&1...); return r.replace(/^.*Warning.*\n?/m, ''); }`
