### Login Failure Root Cause — SQLite WAL Corruption
- **Problem**: Frequent container recreates (`docker compose up -d`) left stale WAL/SHM files that caused password verification to fail on container restart.
- **Solution**: Stop container → delete `database.sqlite-wal` + `database.sqlite-shm` → force `PRAGMA journal_mode=DELETE` → restart.
- **Note on /rest/me**: This endpoint does NOT exist in n8n v2.28.7 (`me.controller.js` has no `Get` decorator). User info is returned directly from login response.
- **Lesson**: Always clean WAL/SHM files when doing direct SQLite password injection.

### Key Lessons
| # | Lesson |
|:-:|--------|
| 195 | **n8n path prefix is MANDATORY behind Cloudflare Tunnel** — n8n v2.28.7 internal REST routes live at `/{N8N_PATH_PREFIX}/rest/...`. Login works at root (`/rest/login`) but protected routes need prefix. |
| 196 | **Python bcrypt ($2b$) ≠ bcryptjs ($2a$) for n8n** — Python's bcrypt library generates hashes with `$2b$` prefix, but n8n's bcryptjs only recognizes `$2a$` hashes. |

| 198 | **Cloudflare Tunnel ignores local config.yml for ingress** — uses remote API-pushed config exclusively. |
| 199 | **PS VIBE dashboard `/assets/` conflicts with n8n** — both serve static assets from `/assets/`. |
| 200 | **n8n v2.28.7 has no `/rest/me` GET endpoint** — `me.controller.js` only has `Patch`, `Post` decorators. User data after login comes from login response body. |
