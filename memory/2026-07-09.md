### Outline API Data Limit Usage
```python
# Set data limit on Outline key
limit_bytes = plan_data_limit_gb * 1024 * 1024 * 1024  # Convert GB to bytes
api_call("PUT", f"/access-keys/{kid}/data-limit", 
         {"limit": {"bytes": limit_bytes}}, api_url=api_url)
```

### New Lessons

129. **Outline data limit = 0 means unlimited** — Outline API treats absent data limit as unlimited. Setting `{"limit": {"bytes": 0}}` also means unlimited. To set a limit, send positive byte count.

130. **`store_key_in_db()` default params — must propagate changes** — When adding a new column to `vpn_keys`, must update both `store_key_in_db()` function signature AND all callers. Missed the admin Handler class outline creation call (not agent), which doesn't pass `outline_tier`.

131. **Sub-tab isolation in agent portal** — Create form for Xray vs Outline must be completely separate sections, not toggled visibility of the same input elements. Different form structure, different hidden fields, different JS. Use `style="display:none"` on sections, not field-level hiding.

132. **Outline doesn't support hybrid server** — Only SG or TH for Outline keys. Dropdown must be filtered in the Outline form.

133. **Outline API PUT /data-limit takes `{"limit": {"bytes": N}}`** — Not `{"bytes": N}`. Nested object.
