
### CRITICAL: SEL Dashboard JS SyntaxError (2026-07-17 12:12 UTC)
- **Root cause:** `esc()` function had `{ {` (double brace) → `function esc(s) { { return ...; }` — missing closing `}` caused `Unexpected end of input` at end of script.
- **Effect:** Entire JavaScript was syntactically invalid → NO JavaScript executed on page load → `switchTab()` never ran → all tabs hidden (empty main content). Only CSS worked (sidebar, modals).
- **Discovery:** Image from Osmo showed empty main content area. Node.js `--check` confirmed SyntaxError. Backup file showed correct version (`{ return ...; }` with single brace).
- **Note:** This bug was pre-existing, NOT introduced by current session's edits. User only noticed after modals were hidden.
- **Lesson #161:** Always run `node --check` after ANY HTML/JS changes — one extra `{` kills entire page.
- **Fix:** Removed extra `{` → `function esc(s) { return ...; }` ✅

## PS VIBE Customer Bot — 6+ Bugs Fixed 🔧

### 1. Booking Route Conflict — `POST /api/bookings` Duplicate Handlers 🐛
- **Problem:** Two `@router.post("/bookings")` handlers in booking_routes.py — Web Dashboard handler (line 444, requires console_id) intercepted requests before Customer Bot handler (line 1468, auto-assigns console). Starlette route registration priority issue.
- **Fix:** Merged into single `api_bookings_create` handler with format detection: checks `customerName` (bot) vs `customer_name` (dashboard/web) → routes accordingly. Web → `status=confirmed`, Bot → `status=pending`.
- **Root commit:** `017c54e` "Deposit lifecycle fix"
- **Lesson #159:** Starlette route registration order matters — duplicate route decorators = first registered wins, not last.

### 2. MySQL Password Mismatch 🔑
- **Fix:** `ALTER USER 'psvibe_user'@'172.17.0.1' PASSWORD ...` to match env `PsVibe@User2024!`
- **Files:** `inventory_fifo.py` + 4 other files had hardcoded old password → updated to `PsVibe@User2024!`

### 3. Booking Visibility — Verified Deposits Disappearing 👻
- **Bug:** `search-bookings` filter (line 1107) only checked `deposit_status = 'paid'` → verified deposits (status='verified') hidden from timeline
- **Fix:** Changed to `deposit_status IN ('paid', 'verified')`
- **Lesson #160:** When adding `deposit_status='verified'` transition, update ALL filters that check for 'paid'.

### 4. Merged Handler Format Detection Bug 🐛
- **Bug:** Customer bot branch had an `elif not is_web_format:` with `else: staff_booking` — but is_web_format was checked after the bot branch, so bot branch always executed for dashboard requests too.
- **Fix:** Moved `is_web_format` check BEFORE bot branch.

### 5. Wallet Insufficient Check Skipped for Members 🐛
- **Bug:** `if booking_id:` block unconditionally set `wallet_mins=None` for ALL booking sessions (guest + member) in `launch_session_sale()`
- **Fix:** Added `if not is_guest:` inside booking_id block
- **Lesson #158:** Booking flows must differentiate guest vs member — guest optimizations need `if not is_guest:` guard.

### 6. Garbled Unicode + Referral Code Fixes
- Removed garbled Burmese text from deposit verify notification
- Fixed referral code circular imports + GSheet→API migration + `fetch_members()` dict parsing
