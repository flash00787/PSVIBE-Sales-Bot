### Fix: Payment History UI crash & Record Payment "Value error"
- **Root cause #1:** `parse_form()` consumes `self.rfile` body stream → JSON POST handler reads empty → JSONDecodeError → "Value error".

- **Root cause #2:** `record_agent_payment()` used `datetime.datetime.utcnow()` but `datetime` was imported as class not module → AttributeError.
  - **Fix:** `datetime.now(dt_timezone.utc)`.
- **Root cause #3:** `/record-payment` POST handler was AFTER general auth check (which sent `send_redirect("/login")`).
  - **Fix:** Moved entire handler block ABOVE auth check.
- **Root cause #4:** Payment History page returned HTML fragment without nav/CSS/JS.
  - **Fix:** Route through `render_keys(payment_history_agent_id=agent_id)` for full admin page layout.
- **Root cause #5:** History → auto switch to first tab (Agents) instead of Payments tab.
  - **Fix:** `switchAdminSubTab` function at bottom of HTML body, called at 200ms after page load.
- **Root cause #6:** `function name conflict` — `switchSubTab()` used for BOTH key sub-tabs (Active/Expired) AND admin sub-tabs (Agents/Pricing/Payments).
  - **Fix:** Admin sub-tabs renamed to `switchAdminSubTab()`.
- **Root cause #7:** Record Payment modal `min="1" step="100"` → 1000 rejected (1000-1 ≠ multiple of 100).

## New Feature: FIFO Key Slot Settlement System (Agent Portal)

### DB Schema
- `commissions` table migration: added `settled INTEGER DEFAULT 0`, `settled_at TEXT`, `settled_by_payment_id TEXT`.

### Helper Functions
- `get_unpaid_commission_count(agent_id)` — counts commissions where `settled = 0`.
- `get_settled_commission_count(agent_id)` — counts commissions where `settled = 1`.
- `get_unsettled_commissions(agent_id)` — returns list of unsettled commissions ordered by `created_at ASC`.
- `settle_commissions_fifo(agent_id, payment_amount, payment_id)` — FIFO settlement engine.
  - For each unsettled commission (FIFO order):
    - `due = base_price - commission` (e.g., 5,000 - 1,000 = 4,000)
    - If `payment_amount >= due`: full settle → `payment_amount -= due` → counter++
    - If `payment_amount < due`: partial settle → remaining due = due - payment_amount → exit
  - Returns: `{settled_count, remaining_amount, fully_settled: bool}`

### Agent Dashboard Updates
- Payment Summary card: Total Sales, Commission (earned), Paid (to admin), Outstanding
- Unpaid Slots: pending commission count / max_keys (e.g., "3 / 10")
- Settled count shown

### check_agent_limits() Changes
- Added: `if unpaid >= max_keys → False, "Unpaid slot limit reached ({}/{})"`.

### Lessons
- #119: **FIFO settlement = reduce unpaid slot count, not total key count** — Key limit now tracks unpaid commissions, not total keys.
- #120: **Partial settlement must update remaining outstanding** — If payment < due, update commission's remaining amount, don't skip.
- #121: **Commission is 20%, not 20% of base_price in all cases** — `due = base_price - commission` where commission = base_price * 0.20.

### Critical Fix: FIFO Only Settles, Never Unsettles 🐛→✅
- **Bug**: `settle_commissions_fifo()` only SETTLES commissions but never UNSETTLES. When a payment is deleted, `total_paid - settled_amount` becomes negative (-32,999), but function breaks on `if remaining >= amount_due` without resetting the commission to unsettled.
- **Fix**: Reset ALL commissions to `settled=0, settled_at=NULL, settled_by_payment_id=NULL` at the start of FIFO, then recalculate from scratch.
- Now properly handles: payment delete (unsettles commissions), payment amount decrease (unsettles proportional), payment amount increase (settles more), new commission (settles FIFO from scratch)
- Function flow: reset all → sort commissions ASC → iterate: if `amount_due <= 0 OR remaining >= amount_due` → SETTLE → `remaining -= amount_due` → else BREAK

### New Lessons (#123-#124)
| # | Lesson |
|:-:|--------|
| 123 | **FIFO must reset before recalc** — If you only loop over `settled=0` commissions, deleting a payment never un-settles. Must reset all to `settled=0` then recalculate from scratch. (#123) |
| 124 | **Payment edit/delete = FIFO trigger** — Any change to agent_payments must trigger FIFO recalculation from scratch. (#124) |

## Late Session (23:00-23:20 UTC) — Earnings Mobile Fix + Proxy Query String Bug 🔥

### Bugs Fixed
1. **📅 Earnings page "This Month"/"Last Month" SQL filter broken** — `date('now', 'start of month', '+1 day', '+6 hours')` was corrupt because `date()` ignores time portion. The `+6 hours` caused July 1 00:00 UTC → June 30 18:00 UTC → still June in date context, so this-month started July 2. Fixed: replaced with `datetime('now', 'start of month')` for This Month and `datetime('now', 'start of month', '-1 month', 'start of month')` / `datetime('now', 'start of month')` for Last Month range. (#131)

2. **🌐 Agent proxy query string not forwarded** — FastAPI route `{path:path}` captures only the path, not query string. Proxy was forwarding to `http://localhost:9357/agent/earnings` without `?range=this-month`. Fixed: appended `str(request.url.query)` to target URL. (#132)

3. **🌐 Proxy `request.url.query.decode()` crashes** — `request.url.query` returns a `URL` property that is a string, not bytes. `bytes.decode()` called on string raises AttributeError → Internal Server Error 502. Fixed: use `str(request.url.query)` instead. (#133)

4. **📱 Agent portal mobile responsive** — Added mobile CSS for earnings page (filter buttons full-width, table horizontal scroll, stat cards compact), dashboard, key list, nav. All improvements in `@media(max-width:640px)`. (#134)

### New Lessons
- #131: SQLite `date()` ignores timezone/`+N hours` — always use `datetime()` for time-adjustable date filters
- #132: FastAPI `{path:path}` route param captures only path without query string
- #133: Starlette `request.url.query` returns a string, not bytes — no `.decode()` needed
- #134: Mobile-friendly CSS for agent portal: table overflow-x:auto, filter button flex:1 on small screens
