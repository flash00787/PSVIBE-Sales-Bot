## Bug Fixes — VPN Agent System (Admin Handler)

### Fix: Payment History UI crash & Record Payment "Value error"
- **Root cause #1:** `parse_form()` consumes `self.rfile` body stream → JSON POST handler reads empty → JSONDecodeError → "Value error".
  - **Fix:** Save `self._raw_body` before `parse_form()`, replace `self.rfile` with `BytesIO(self._raw_body.encode())` for form parsing. JSON handlers read from `self._raw_body`.
- **Root cause #2:** `record_agent_payment()` used `datetime.datetime.utcnow()` but `datetime` was imported as class not module → AttributeError.
  - **Fix:** `datetime.now(dt_timezone.utc)`.
- **Root cause #3:** `/record-payment` POST handler was AFTER general auth check (which sent `send_redirect("/login")`).
  - **Fix:** Moved entire handler block ABOVE auth check.
- **Root cause #4:** Payment History page returned HTML fragment without nav/CSS/JS.
  - **Fix:** Route through `render_keys(payment_history_agent_id=agent_id)` for full admin page layout.
- **Root cause #5:** History → auto switch to first tab (Agents) instead of Payments tab.
  - **Fix:** `switchAdminSubTab` function at bottom of HTML body, called at 200ms after page load.
- **Root cause #6:** `function name conflict` — `switchSubTab()` used for BOTH key sub-tabs (Active/Expired) AND admin sub-tabs (Agents/Pricing/Payments).
  - **Fix:** Admin sub-tabs renamed to `switchAdminSubTab()`.
- **Root cause #7:** Record Payment modal `min="1" step="100"` → 1000 rejected (1000-1 ≠ multiple of 100).
  - **Fix:** Changed to `min="100" step="100"`.

### Fix: Agent badge missing on admin key cards
- `agent_id` missing from merged_outline and merged_xray dicts in `render_keys()`.
- **Fix:** Added `agent_id: dbk.get("agent_id", "")` to both merge dicts.

### Fix: Active/Expired sub-tab "Value error"

## New Feature: FIFO Key Slot Settlement System (Agent Portal)

### DB Schema
- `commissions` table migration: added `settled INTEGER DEFAULT 0`, `settled_at TEXT`, `settled_by_payment_id TEXT`.

### Helper Functions
- `get_unpaid_commission_count(agent_id)` — counts commissions where `settled = 0`.
- `get_settled_commission_count(agent_id)` — counts commissions where `settled = 1`.
- `get_unsettled_commissions(agent_id)` — returns list of unsettled commissions ordered by `created_at ASC`.
- `settle_commissions_fifo(agent_id, payment_amount, payment_id)` — FIFO settlement engine.
  - For each unsettled commission (FIFO order):
    - `due = base_price - commission` (e.g., 5,000 - 1,000 = 4,000)
    - If `payment_amount >= due`: full settle → `payment_amount -= due` → counter++
    - If `payment_amount < due`: partial settle → remaining due = due - payment_amount → exit
  - Returns: `{settled_count, remaining_amount, fully_settled: bool}`

### Agent Dashboard Updates
- Payment Summary card: Total Sales, Commission (earned), Paid (to admin), Outstanding
- Unpaid Slots: pending commission count / max_keys (e.g., "3 / 10")
- Settled count shown

### check_agent_limits() Changes
- Added: `if unpaid >= max_keys → False, "Unpaid slot limit reached ({}/{})"`.

### Lessons
- #119: **FIFO settlement = reduce unpaid slot count, not total key count** — Key limit now tracks unpaid commissions, not total keys.
- #120: **Partial settlement must update remaining outstanding** — If payment < due, update commission's remaining amount, don't skip.
- #121: **Commission is 20%, not 20% of base_price in all cases** — `due = base_price - commission` where commission = base_price * 0.20.
