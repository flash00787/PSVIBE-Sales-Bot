## Session (2026-07-12, ~19:50 UTC) — Deposit Wrong Amount Root Cause Fix 🔧

### Bug: Bot Asks 3,000 Ks Instead of 7,000 Ks 🐛→✅

**Reported by:** Osmo ("Deposit တောင်းတဲ့ အမောင့်က မှားနေတာ")

**Symptom:** Booking #1613 (PS5 Pro, 2hrs, 12,000/hr) → bot shows **3,000 Ks** deposit instead of correct **7,000 Ks**.

**Root Cause — `resp.success` AttributeError on dict:**
- `_calc_deposit_amount()` calls API → gets correct response `{"deposit_amount": 7000, "session_fee": 24000, ...}`
- `_validate_response()` runs `unwrap_response()` which extracts inner data → returns DICT, not object
- Code checks `resp.success and resp.data` (attribute-style on a dict) → **AttributeError** caught by `except`
- Falls back to hardcoded fallback: 5,000/hr × 2hr × 30% = **3,000 Ks** ❌
- The API call was working fine — the code just couldn't read the result!

**Fix Applied:**
- **File:** `/root/psvibe-sales-bot/customer_bot/booking_handlers.py` (~line 2715)
- Changed: `resp.success and resp.data and resp.data.get("deposit_amount")`
- To: `resp and isinstance(resp, dict) and resp.get("deposit_amount")`
- **Bot restarted:** `sudo systemctl restart psvibe-customer-bot` ✅

**Correct deposit for PS5 Pro 2hrs:**
- Base rate: 10,000 Ks/hr
- PS5 Pro multiplier: 1.2
- Session fee: 2 × 10,000 × 1.2 = 24,000 Ks
- Deposit (30%): 7,200 Ks → rounded to nearest 500 = **7,000 Ks** ✅

### Lesson (#160)
| # | Lesson |
|:-:|--------|
| 160 | Bot's `_api_get` → `_validate_response` → `unwrap_response` returns **dict** (inner data), not object. Never use `resp.success` attribute-style access on API responses — use `isinstance(resp, dict) and resp.get(...)` instead. |

- `settings_config` has correct `console_multipliers: {"C-09": 1.2}` (no spaces in key)
