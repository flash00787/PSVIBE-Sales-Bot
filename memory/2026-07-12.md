## Session (2026-07-12) — AKT Clothing Payment System Bugs 🔧

### Overview
Worked with Osmo on AKT Clothing Shop payment system. Fixed multiple bugs related to payment recording, account selection, and edit functionality.

### Bugs Found & Fixed ✅

#### Bug 1: Purchase Payment Double-Negation → Balance တက်သွားတာ
- **File:** `/opt/kora-projects/akt_clothing/code/backend/app/routes/purchase_routes.py`
- **Root cause:** `_record_payment_transaction_mongo()` applies `-amount` for `"purchase"` type, but `create_purchase` was passing already-negative values → double negation = balance INCREASED
- **Fix:** Changed `-paid_amount` → `paid_amount` (pass positive, let function negate)

#### Bug 2: Single Payment မှာ Account Dropdown မပါတာ
- **File:** `/opt/kora-projects/akt_clothing/code/frontend/src/views/Purchases.vue`
- **Root cause:** New purchase form single-payment mode only had `paid_amount` input, no account selection
- **Fix:** Added `paymentAccount` ref + dropdown (payment accounts list), sent as `payment_breakdown` on submit

#### Bug 3: Edit Purchase Payment Transaction မမှတ်တာ
- **File:** `/opt/kora-projects/akt_clothing/code/backend/app/routes/purchase_routes.py`
- **Root cause (3 sub-issues):**
  1. `delete_many()` used wrong field names (`ref_type`/`ref_id` → `reference_type`/`reference_id`) → old transactions never deleted
  2. `_record_payment_transaction_mongo` hardcoded to `'Cash Register'` for non-split edit
  3. Frontend `saveEdit()` didn't send `payment_breakdown` for single-payment mode
- **Fix:** Fixed field names, frontend sends account data on edit

#### Bug 4: R/P Pay Supplier — Payment Method Redundant
- **File:** `/opt/kora-projects/akt_clothing/code/frontend/src/views/RPayables.vue`
- **Root cause:** Had BOTH "Payment Method" dropdown (Cash/KPay/WavePay/Bank Transfer) AND "Transfer Account" dropdown (separate) — redundant since account implies method
- **Fix:** Replaced Payment Method with actual payment accounts list, removed Transfer Account section. Split rows also use account names.
