## Fix 7 (NEW): Food Cart Feature — Session Food Orders (17:44 UTC)
**Request:** Customers need to order food during gaming sessions, payment at session end.
**Design:** Boss approved Phase 1 → Phase 2 approach. Phase 1 done today.
**Status:** ✅ Deployed and tested

### What was built:

#### 1. MySQL `food_cart` Table
```sql
id INT AUTO_INCREMENT PRIMARY KEY,
booking_id VARCHAR(50) NOT NULL,
item_name VARCHAR(255) NOT NULL,
quantity INT NOT NULL DEFAULT 1,
unit_price DECIMAL(10,0) NOT NULL,
created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
fulfilled_at DATETIME DEFAULT NULL
```

#### 2. API Endpoints (`/root/psvibe_api_server/patch_routes.py`)
| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/food-cart` | POST | Add item to cart |
| `/api/food-cart/{booking_id}` | GET | Get unfulfilled items |
| `/api/food-cart/{booking_id}` | DELETE | Mark as fulfilled (after sale completed) |
| `/api/food-cart/{booking_id}/cancel` | DELETE | Remove unfulfilled items |

#### 3. Bot — Console "Food Sale" Button (`console.py`)
- New button `BTN_FOOD_SALE` added to Console Management keyboard
- When clicked → shows list of active sessions → staff picks one via `/_console_session_food_pick`
- Calls `cmd_session_food_order(target)` which:
  - Pre-fills member/console from session data
  - Sets `session_food_order=True` flag
  - Opens `prompt_food_menu`

#### 4. Bot — Session Food Order (`sales.py`)
- New function `cmd_session_food_order(update, context, target)` — sets up user_data with session info
- New function `_save_food_cart(update, context)` — intercepts `BTN_DONE` when flag is set:
  - Saves items to food_cart via `POST /api/food-cart`
  - Returns to Console Menu (no payment)
- Modified `step_food_menu`: checks `session_food_order` flag at `BTN_DONE`

#### 5. Bot — Auto-Load at Session End (`sales.py` `launch_session_sale`)
- After `context.user_data.update({...})`, fetches `GET /api/food-cart/{booking_id}`
- Adds all cart items to `context.user_data["food_items"]` automatically
- Items appear in Daily Sales form when voucher opens

### Flow:
```
Console → [🍔 Food Sale] → Pick console → Food Menu → Items → Done
  → Cart saved (no payment) → Console Menu
  
Session End → Booking ends → Voucher opens → Food cart items auto-loaded
  → Game Time + Food = Total → Payment (Cash/KPay/Wave/Aya Pay)
```

| `/root/psvibe-sales-bot/bot/handlers/sales.py` | cmd_session_food_order, _save_food_cart, food_cart auto-load in launch_session_sale |

## Fix 8: Import Path Error — `cmd_session_food_order` (18:20 UTC)
**Error:** "Food Sale" button shows active consoles → staff picks console → bot shows game list (wrong menu) / crashes
**Root cause:** `cmd_session_food_order` (sales.py line 145) imports `_psvibe_get_async` from `bot.api_client`, but the function is defined in `bot/__init__.py` (line 2585). Import silently provided the function, but at runtime the wrong or broken import caused:
- `fetch_members_async = api_fetch_members_async` alias at `__init__.py` line 2555 which returns a list of member dicts, not a list of IDs
- `cmd_session_food_order` calls `await fetch_members_async()` → gets dicts → passes to `prompt_food_menu` which expects IDs → breaks

**Fix:** Changed 5 occurrences of `from bot.api_client import _psvibe_get_async` → `from bot import _psvibe_get_async` in sales.py
**Duration:** Broken for ~1 hour before root cause found

### Fix 9: BTN_CLEAR_CART Dead Code (18:20 UTC)
**Error:** BTN_CLEAR_CART handler never triggers
**Root cause:** In `step_food_menu`, when `session_food_order` flag is set and user clicks Done, the code hits `return await prompt_confirm(...)` BEFORE the `BTN_CLEAR_CART` handler, making it unreachable dead code
**Fix:** Restructured BTN_CLEAR_CART handler to come before the BTN_DONE block

### Fix 10: Coupon Generation Blocking — Voucher Slow to Open (18:34 UTC)
**Error:** Sale voucher opening still very slow (Boss confirms)
**Root cause:** `step_end_session` in `console.py` line 370 has `await asyncio.to_thread(api_post, "coupons/generate", ...)` — this BLOCKS the entire session end flow until the coupon API responds. The earlier fix using `asyncio.create_task()` was reverted by git stash/patch.
**Fix:** Wrapped coupon generation in `async def _gen_coupon_async()` and called via `asyncio.create_task(_gen_coupon_async())` — no longer blocks voucher opening.
**Status:** ✅ Deployed, psvibe-sale-bot restarted
