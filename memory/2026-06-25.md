## 3. 🐛 Customer Bot — Phone Last-3-Digits Member Lookup Fix ✅

### Bug
Customer Booking flow: "Member Card ရှိတယ်" → ဖုန်းနောက်ဆုံး ၃ လုံးရိုက် → **member ရှာမတွေ့**

### Root Cause
After MySQL migration, `api_fetch_members` endpoint now returns **full member objects** (`[{id, name, phone, ...}]`) instead of **flat ID list** (`["PSV-001", ...]`). Customer Bot's `_fetch_members()` treated dict objects as string IDs → invalid API URLs → silent failure → empty member list.

### Fix (2 files)
| File | Change |
|------|--------|
| `customer_bot/api.py:297-372` | `_fetch_members()` — detect format: if first item is dict → build members directly (1 API call instead of 1+N) |
| `customer_bot/booking_handlers.py:1160` | Balance lookup: `wallet_mins` → `balance_mins` → `balance` fallback chain |

### Bonus
Member fetch speed improved: 1 API call instead of 1 list + 7 detail calls.
