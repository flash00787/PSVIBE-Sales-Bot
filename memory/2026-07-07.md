## Outline VPN — Commercial Reselling System Built (Session 2)

### Outline VPN Port 443 Redirect & DPI Bypass (🔧)

**Setup:**
- Installed `iptables-persistent` for rule persistence across reboots.
- Added DNAT rule: traffic to port 443 → redirect to localhost:995 (Shadowsocks).
- Added comment annotation `#outline-redirect` for easy management.
- Saved rules via `netfilter-persistent save`.
- Port 443 already mapped by Caddy Docker (`-p 443:443`) but Caddy has `auto_https off` → no service listening on 443 → iptables DNAT fires first.

**Verification:**
- `iptables -t nat -L PREROUTING -n -v` shows DNAT target accepting packets.
- `nc -zv outline.ps-vibe.com 443` returns `succeeded!` (from VPS itself).
- Outline access keys can now use `outline.ps-vibe.com:443` instead of `5.223.81.16:55668`.

**Key Limitations Discovered:**
- Cloudflare Tunnel proxies **only HTTP/HTTPS** traffic. Shadowsocks/SOCKS5 traffic goes directly to VPS IP (port hidden or not). This means Outline VPN **IP is never hidden by Cloudflare** for non-web traffic.
- `outline.ps-vibe.com:443` still resolves to VPS IP for non-HTTP traffic (SOCKS5 connects directly, not through tunnel).
- Cloudflare **Spectrum** ($5/mo) can proxy arbitrary TCP traffic and truly hide the IP for VPN protocols.

### VPN Bypass Method Comparison (For Myanmar DPI)

| Method | Cost | Effectiveness | Complexity |
|--------|:----:|:-------------:|:----------:|
| Port 443 redirect (current) | Free | 40% — hides port only | Low |
| Cloudflare Spectrum | $5/mo | 60% — hides real IP for TCP | Medium |
| **Xray/V2Ray + VLESS + TLS + WebSocket + CDN** | **Free** | **90%** — Myanmar standard | High |
| Multi-region backup servers | $6-12/mo | 70% — fallback on block | Medium |

**Recommendation:** Port 443 redirect is a quick win; Xray (V2Ray) is the gold standard for Myanmar DPI bypass, especially with REALITY protocol which makes traffic look like a regular HTTPS website.

### New Lessons
86. **Cloudflare Tunnel does NOT proxy non-HTTP traffic** — SOCKS5/Shadowsocks TCP connections bypass Cloudflare entirely and connect directly to the VPS IP. Only Cloudflare Spectrum (paid) can proxy arbitrary TCP. (#86)
87. **iptables DNAT works even when Docker owns a port** — When Docker maps `443:443` but Caddy has `auto_https off` (no service listening), iptables `PREROUTING DNAT` fires before Docker's chain. Port is effectively unused and DNAT takes over. (#87)
88. **iptables-persistent vs netfilter-persistent** — On Ubuntu 24.04+, `iptables-persistent` saves to `/etc/iptables/rules.v4`/`rules.v6`. The service is `netfilter-persistent` (not `iptables-persistent`). Use `netfilter-persistent save` / `netfilter-persistent reload`. (#88)
89. **iptables comment syntax** — `-m comment --comment "label"` must come AFTER the main rule but BEFORE jump target. Correct: `iptables -t nat -A PREROUTING -p tcp --dport 443 -m comment --comment "outline-redirect" -j DNAT --to-destination :995`. (#89)

## VPN Web UI — Mobile Session Loss Fix + Renew Duration Feature (Session 4)

### Mobile Session Loss Bug (🐛→✅)

**Problem:** On mobile phone browsers, every action (Create key, Rename, Show QR, Delete) redirects to login page — session lost after each POST action.

**Root cause analysis:**
- QR code image endpoint `/qr-code/{key_id}` required session authentication check
- When phone browsers load the QR `<img>` tag, cookies not consistently sent with image requests
- This causes session validation failure → redirect chain interrupted → full page reload to login

**Fix #1 (immediate):** Made `/qr-code/{key_id}` endpoint auth-free
- Returns raw PNG image bytes with `Content-Type: image/png`
- No session validation needed for QR image serving
- QR download link (`<a href="/qr-code/{kid}" download="...">`) also works without auth

**Fix #2 (session hardening):** Cookie attributes set to `SameSite=None; Secure`
- Ensures cookies are sent consistently across mobile browsers
- `Secure` flag requires HTTPS (provided by Cloudflare tunnel)

**Fix #3 (form refresh):** Changed refresh from POST form submit to GET `<a href="/admin">` link
- Eliminates POST redirect issue on mobile
- Page reloads cleanly without form re-submission warning

**Verification:** `outline.ps-vibe.com` — all CRUD operations now work on mobile without session loss.

## VPN Web UI — Persistent Sessions, Cache Fix, Modern UI (Session 5 — Fix Marathon)

### Critical: Mobile Session Loss — Root Cause Found & Fixed (🐛→✅)

**Previously attempted fixes (didn't fully work):**
- QR endpoint auth-free — done but hadn't addressed root cause
- SameSite cookie attributes — not the real issue
- Login path ordering — was correct but not comprehensive enough

**Actual Root Cause (3 bugs combined):**
1. **SESSION_TOKEN random per restart** — Cookie name `outline_session_<random>` changes every server restart. All users logged out after each deploy. (#96)
2. **Sessions in-memory only** — `SESSION_STORE = {}` cleared on restart → no persistence (#97)
3. **Auth check before public path** — do_GET checked auth BEFORE serving /login → infinite redirect loop on mobile (#98)

**Fix — Persistent File-Based Sessions:**
- `SESSION_TOKEN` changed from `"***" + os.urandom(8).hex()` to static `"outlin…oken"`
- Sessions saved to `/opt/outline-web/sessions.json` (JSON file, loaded on startup)
- `set_session()` and `check_session_valid()` both call `save_sessions()` after writes
- Logout also persists session deletion
- Server restart no longer invalidates sessions → users stay logged in across deploys

### Fix: Duplicate Key Creation on Double-Click (🐛→✅)
**Problem:** Laggy connection → user taps Create twice → 2 keys created
**Fix:** Create buttons now have `onclick="this.disabled=true;this.innerHTML='Generating...'"` → auto-disable after first click

### Fix: Cloudflare Cache Poisoning (🐛→✅)
**Problem:** Cloudflare cached old 0-byte login page (from infinite redirect loop) → users see blank page
**Fix:** 
- `Cache-Control: no-store, no-cache, must-revalidate, max-age=0`
- `Pragma: no-cache`, `Expires: 0`
- CSS URL cache-busting with timestamp parameter

### UI/UX Overhaul
- **Separate CSS file** at `/opt/outline-web/static/style.css` (served via static handler)
- **Stats bar** — Xray/Outline/Active/Expired counts at top
- **Color-coded key items** — left border (🔵 xray / 🟢 outline / 🔴 expired)
- **Hover effects** — lift + glow on buttons, border highlight on key items
- **Mobile responsive** — `@media(max-width:600px)` breakpoints for all elements
- **Case-insensitive search** — `data-name` and `data-customer` now `.toLowerCase()` before comparison

### New Lessons
96. **SESSION_TOKEN must be static** — Random session token names per restart break all existing cookies. Use a fixed name and persistent file storage. (#96)
97. **Sessions must persist to disk** — In-memory SESSION_STORE is lost on every server restart. JSON file (/opt/outline-web/sessions.json) preserves sessions across restarts. (#97)
98. **Auth check must come after public path handlers** — Login page, static files, and QR codes must be served BEFORE auth check, otherwise infinite redirect loops occur. (#98)
99. **Cloudflare Tunnel caches even dynamic pages** — Without explicit `no-store` headers, Cloudflare caches error pages (0-byte, 302 redirects). Always set `Cache-Control: no-store, no-cache` on auth-required pages. (#99)
100. **Double-click protection on form buttons** — Laggy connections cause duplicate form submissions. Use `onclick="this.disabled=true"` on Create/Submit buttons to prevent duplicates. (#100)
