## 🐛 Bug Fixes (05:00–05:27 UTC, 11:30 AM–12:00 PM MMT)

### Fix 1: Session Timer Drift on Bot Restart → end-of-session
**Bug:** Bot restart တိုင်း timer က 2h+ drift သွားတာ — e.g., 8100s delay after restart
**Root cause:** `_end_dt_iso = now + planned_mins` — `now` = restart time (NOT original start time)
**Fix:** Calculate `end_t` from stored HH:MM format + dual-layer protection:
- Layer 1: `restore_session_reminders()` → `_active_timers` recovery uses `end_t.time()`
- Layer 2: `start_session_reminder()` → validates drift < 300s tolerance, recalculates if drifted
**Files:** `session_reminder_store.py`
**Status:** ✅ Verified — 5 consecutive restarts, max drift 12s

### Fix 2: Console Status start_time Reset by _sync_console_status()
**Bug:** Booking operations (checkin, cancel) → sync → `start_time=NOW()` resets timer on Active consoles
**Root cause:** `_sync_console_status()` always did `start_time=NOW()` — no check if already Active
**Fix:** Check if console already Active → skip `start_time` update. Only set `start_time=NOW()` for newly activated consoles.
**Files:** `psvibe_api_server/app.py` — `_sync_console_status()`
**Status:** ✅ Verified — extend + checkin + cancel all preserve start_time

### Fix 3: Existing Active Consoles start_time Restore
**Same root cause as Fix 2.** Applied fix to all already-active consoles in DB.
**Status:** ✅ All active consoles verified
