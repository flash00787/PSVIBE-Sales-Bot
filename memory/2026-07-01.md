### Root Causes Found

| # | Bug | Root Cause | Fix |
|---|-----|-----------|-----|
| 1 | BS Member Liability = 0 | `ym` variable undefined in `get_balance_sheet()` — `NameError` silently swallowed by `except: _fr2 = {"liability": 0}` | Replaced `ym` with `_month_key` variable |
| 2 | Cashflow investing section cumulative | `_ap`, `_pp`, `_dp`, `_adv_total` queries had no month filter — showed all-time totals every month | Added `created_at >= _start AND < _end` to all 4 queries |
| 3 | Cashflow closing wrong (-19M) | Formula `closing = opener + op_net + inv_net` missed 300M capital + 30M capital injects in cash_movements | Changed to cumulative SQL query for both opener and closing (same formula, different time boundaries) |
| 4 | Cashflow closing 30M too high | `transfer_out` amounts stored as negative in DB; `- SUM(transfer_out)` double-subtracted | Changed to `+ SUM(transfer_out)` (adding negative = subtracting) |
