### Path 1: Customer Creates Booking (via Customer Bot)

- **Customer receives confirmation:** ❌ **NO.** No "သင်၏ Booking ကို အတည်ပြုပြီးပါပြီ" message.
- **Admin group notified:** ✅ **Yes.** `_notify_booking_received()` sends to hardcoded chat ID `-1003686032747`.
- **GAP:** Customer receives NO acknowledgment.

### Path 2: Staff Approves → Customer Notified

- **Customer notification:** ✅ **YES.** Sends confirmation message.
- **Admin notification:** ✅ **YES.** Inline card edited.
- **Reminders scheduled:** ✅ **YES.** n8n webhook and bot fallback.

### Path 3: Staff Rejects → Customer Notified

- **Customer notification:** ✅ **YES.** Sends rejection message.
- **Reminders cancelled:** ✅ **YES.**

### Path 4: Staff Cancels → Customer Notified

- **Customer notification:** ✅ **YES.** Sends cancellation message.
- **Admin notification:** ❌ **NO separate admin group notification.** Only inline card edited.

### Path 5: Customer Self-Cancels → Customer Notified

- **`_text_cancel_booking()`:** ✅ Customer notified, ✅ Admin notified, ❌ Reminders NOT cancelled.
- **`cmd_cancel_booking()`:** ✅ Customer notified, ❌ Admin NOT notified, ✅ Reminders cancelled.
- **GAP:** Inconsistent admin notification and reminder cancellation.

### Path 6: Auto-Cancel / Expired / No-Show

- **Customer notification (API cancel):** ✅ **YES.** (if `telegram_chat_id`)
- **Admin notification:** ❌ **NO.**
- **CRITICAL GAP:** No Proactive No-Show Auto-Cancel. No background job or cron automatically cancels bookings for no-shows.

### Path 7: Booking Checkin

- **Admin notification:** ✅ **YES.** Sends "✅ Customer Checked In!"
- **Customer notification:** ✅ **YES.** Sends "🎉 Welcome to PS VIBE!"
- **No-Show Auto-Cancel:** ❌ No auto-cancel mechanism exists.

---

### Critical Gaps Summary

1.  **❌ Customer receives NO confirmation after booking creation** — Customer bot does not send "သင်၏ Booking ကို လက်ခံပြီးပါပြီ" after form submission.
2.  **❌ No proactive no-show auto-cancel** — No background job automatically cancels bookings when the booking time passes without check-in.
3.  **❌ Inconsistent admin notifications on cancel** — `cmd_cancel_booking` (Path 5b) and `_do_cancel_booking` (Path 4 from staff) don't notify the admin group, only edit the inline card.
4.  **❌ `_text_cancel_booking` doesn't cancel reminders** — Uses a different endpoint (`PUT /api/cancel_booking/{id}`) that doesn't trigger the reminder cancellation webhook.
5.  **⚠️ `_sbk_advance_reminder` sends reminders at 30 min, not 10 min** — The docstring says 10 min but code shows `timedelta(minutes=30)`.
6.  **⚠️ Hardcoded admin chat ID** — `_notify_booking_received` uses hardcoded `-1003686032747` instead of `STAFF_NOTIFY_CHAT` env var.
7.  **⚠️ Double reminder on approve** — Both n8n webhook AND `_sbk_advance_reminder` schedule 30-min reminders, potentially sending duplicate notifications.
