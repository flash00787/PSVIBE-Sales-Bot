# Kora Memory Upgrade — Phase 2 Spec

## Goal
Real-time sub-agent tracking, orphan detection, live dashboard, automated memory consolidation.

## Files to Create

### 1. `memory/active_tasks.json` — Real-time Task Registry
Lightweight JSON with currently-active sub-agents. Updated by Kora before/after spawn.

```json
{
  "version": 1,
  "active": {
    "<task-id>": {
      "taskName": "...",
      "model": "provider/model",
      "spawned": "ISO-8601",
      "task": "full task description",
      "sessionKey": "current-session-key",
      "channel": "telegram",
      "goal": "what it's supposed to accomplish"
    }
  }
}
```

### 2. `memory/subagent_ctl.py` — Master Control Script

Swiss-army knife for sub-agent lifecycle:

**Commands:**
- `register <taskName> <model> <goal>` — Register pre-spawn, outputs task-id
- `complete <task-id> <status> <summary>` — Mark complete, remove from active
- `list` — List all active tasks (handles empty registry gracefully)
- `status` — Quick dashboard (counts + warnings for long-running)
- `orphans` — Find tasks in journal that are "completed"/"partial" but active_tasks still has them (stale entries), or tasks that have been "running"/"spawned" for >30min without completion in the journal
- `summary` — Generate human-readable agent status markdown

**Requirements:**
- Uses only stdlib
- Handles missing files gracefully (empty state)
- Validates JSON integrity
- All commands exit 0 on success, 1 on error

### 3. `memory/agent-status.md` — Live Dashboard
Auto-generated by `subagent_ctl.py summary`. Human-readable markdown.

**Format:**
```markdown
# Agent Status Dashboard
*Generated: 2026-05-28 09:15 UTC*

## 🟢 Active Tasks (3)
| Task | Model | Running For | Goal |
|------|-------|-------------|------|
| create-feature-x | deepseek/deepseek-v4-pro | 2m | Implement X feature |
| fix-bug-y | anthropic/claude-sonnet-4 | 5m | Fix Y bug |

## ⏳ Recent Tasks (last 24h)
- ✅ task-a — completed 10m ago
- ❌ task-b — failed 30m ago (retry available)
- 🟡 task-c — partial 1h ago
```

Also add `upgradedToV2: true` to metadata.

### 5. `memory/boot_protocol.py` — Enhanced
- Read both journal AND active_tasks.json
- Detect orphans: tasks in active_tasks that are NOT in journal as running
- Detect stale: tasks in journal as "running" but spawned >30min ago
- Output machine-parseable JSON summary
- `--json` flag for machine-readable output

### 6. `memory/consolidator.py` — Enhanced to Auto-Write
- `--apply` flag: Actually updates MEMORY.md with distilled summaries
- `--dry-run` flag: Preview only (default behavior)
- Dedup: Don't add content already in MEMORY.md
- Format: `## Memory (YYYY-MM-DD)` section in MEMORY.md

### 7. `memory/heartbeat_routine.py` — Enhanced
- Also check active_tasks.json for stale entries
- Auto-generate agent-status.md on heartbeat
- Report to Boss via summary

## Behavior Rules

### Spawn Protocol (Kora must follow):
1. Generate task-id: `f"{int(time.time())}-{taskName[:20]}"`
2. Write to `subagent-journal.json` with status "running"
3. Write to `active_tasks.json`
4. Call `sessions_spawn`
5. On completion: write to journal + remove from active_tasks

### Cleanup:
- If active_tasks.json has entries that aren't in journal → orphaned → report
- If journal has "running" entries older than 30 min → stale → report
- If agent-status.md hasn't been updated in 1h → stale dashboard
