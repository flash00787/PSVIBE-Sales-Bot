## 🔧 Sale Bot Fixes (12:02-12:19 UTC)

### 1. AYA Pay Missing from Constants
- **Bug:** AYA Pay disappeared from payment options after `apply_fixes.py` run / bot restart
- **Root Cause:** `PAY_METHODS` constant exists in BOTH `bot/constants.py` AND `apply_fixes.py`. Modifying only one leaves the other stale.
- **Fix:** Added "AYA Pay" to both files' PAY_METHODS
- **Files:** `bot/constants.py` L35, `apply_fixes.py`
- **Status:** ✅ Confirmed working

### 2. Session Extended Reminder — Missing message_thread_id
- **Bug:** After session extend → no reminder notification sent
- **Root Cause:** `_do_extend()` calls `notify_update()` without passing `message_thread_id`
- **Fix:** Thread `message_thread_id` through extend call chain
- **Files:** `booking_flow.py`, `console.py`, `sales.py`, `session_reminder_store.py`
- **Status:** ✅ Confirmed working

### 3. Session Reminder — "Plan: 1 min" After Extend
- **Bug:** Reminder shows wrong remaining time after extension
- **Root Cause:** `rem_mins = max(1, int(total_rem_secs / 60))` calculates remaining seconds from end time instead of accumulated duration
- **Fix:** Added `_SESSION_TOTAL_MINS` dict tracking accumulated total minutes
- **Files:** `session_reminder_store.py`
- **Status:** ✅ Confirmed working

### 5. Customer Bot — Slot Availability Date Filter (14:47-14:53 UTC)
- **Bug:** Customer Bot က slot availability စစ်တဲ့အခါ **ရက်စွဲမရွေး** — အားလုံး bookings တွေကို unavailable လုပ်ပြီး "No consoles available" ပြတယ်
- **Root Cause:** `GET /api/search-bookings` endpoint မှာ `date` parameter **လုံးဝမရှိဘူး**။ Client က `date=2026-06-17` ပေးပေမယ့် API က ignore လုပ်တယ် → SQL query က ရက်စွဲမခွဲဘဲ booking အားလုံးကို ပြန်ပေးတယ် → slot logic က အကုန် unavailable လုပ်လိုက်တယ်
- **Fix:** Built dynamic WHERE clause — `date` parameter ပါမှ `booking_date=%s` filter ထည့်; date format 5 မျိုး (YYYY-MM-DD, MM/DD/YY, etc.) ကို normalize လုပ်
- **ရလဒ်:** `date=2026-06-17` → 1 booking for June 17 only ✅ (before fix: 223 bookings across 12 dates)
- **Files:** `psvibe_api_server/app.py` — `api_search_bookings()` endpoint
- **Status:** ✅ Confirmed working

## 🔧 Staff Booking Loop Fix (16:59 UTC / 23:29 MMT)

### 6. Staff Booking (New Booking) — Infinite Loop
- **Bug:** `/newbooking` flow က loop ပတ်နေတယ် — Date → Time → Console → Name → Phone → **Date ပြန်မေး →** Time → Console → Name → Phone → ဒီအတိုင်း အဆုံးမဲ့
- **Root Cause:** `step_sbk_date()` (state `SBK_DATE`) က Phone number လက်ခံပြီးတာနဲ့ **`return SBK_TIME`** လုပ်ပြီး Date ကို ပြန်မေးတယ် — ဒါက previously selected date ကို overwrite လုပ်ပြီး loop ဖြစ်စေတယ်
- **Fix:** `return SBK_TIME` ကို duration keyboard နဲ့ `return SBK_GAME` ပြောင်း — phone ရပြီဆိုရင် duration → game → confirm → create booking ကို ဆက်သွား
- **File:** `bot/handlers/booking.py` — `step_sbk_date()` function
- **Status:** ✅ Fixed + deployed
- Session Extend 404 fix: `_do_extend()` called `_psvibe_post_async("/api/bookings/extend-duration")` — double /api/ in URL path (all other calls use bare path like "bookings/checkin"). Changed to "bookings/extend-duration". Bot restarted ✅
