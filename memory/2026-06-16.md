### 5. Customer Bot — Slot Availability Date Filter (14:47-14:53 UTC)
- **Bug:** Customer Bot က slot availability စစ်တဲ့အခါ **ရက်စွဲမရွေး** — အားလုံး bookings တွေကို unavailable လုပ်ပြီး "No consoles available" ပြတယ်
- **Root Cause:** `GET /api/search-bookings` endpoint မှာ `date` parameter **လုံးဝမရှိဘူး**။ Client က `date=2026-06-17` ပေးပေမယ့် API က ignore လုပ်တယ် → SQL query က ရက်စွဲမခွဲဘဲ booking အားလုံးကို ပြန်ပေးတယ် → slot logic က အကုန် unavailable လုပ်လိုက်တယ်
- **Fix:** Built dynamic WHERE clause — `date` parameter ပါမှ `booking_date=%s` filter ထည့်; date format 5 မျိုး (YYYY-MM-DD, MM/DD/YY, etc.) ကို normalize လုပ်
- **ရလဒ်:** `date=2026-06-17` → 1 booking for June 17 only ✅ (before fix: 223 bookings across 12 dates)
- **Files:** `psvibe_api_server/app.py` — `api_search_bookings()` endpoint
- **Status:** ✅ Confirmed working

## 🔧 Staff Booking Loop Fix (16:59 UTC / 23:29 MMT)

### 6. Staff Booking (New Booking) — Infinite Loop
- **Bug:** `/newbooking` flow က loop ပတ်နေတယ် — Date → Time → Console → Name → Phone → **Date ပြန်မေး →** Time → Console → Name → Phone → ဒီအတိုင်း အဆုံးမဲ့
- **Root Cause:** `step_sbk_date()` (state `SBK_DATE`) က Phone number လက်ခံပြီးတာနဲ့ **`return SBK_TIME`** လုပ်ပြီး Date ကို ပြန်မေးတယ် — ဒါက previously selected date ကို overwrite လုပ်ပြီး loop ဖြစ်စေတယ်
- **Fix:** `return SBK_TIME` ကို duration keyboard နဲ့ `return SBK_GAME` ပြောင်း — phone ရပြီဆိုရင် duration → game → confirm → create booking ကို ဆက်သွား
- **File:** `bot/handlers/booking.py` — `step_sbk_date()` function
- **Status:** ✅ Fixed + deployed
- Session Extend 404 fix: `_do_extend()` called `_psvibe_post_async("/api/bookings/extend-duration")` — double /api/ in URL path (all other calls use bare path like "bookings/checkin"). Changed to "bookings/extend-duration". Bot restarted ✅

## 🔧 Session Total Mins Persistence (17:51 UTC)

### 7. _SESSION_TOTAL_MINS → survives bot restart
- **Bug:** Bot restart ဖြစ်ရင် `_SESSION_TOTAL_MINS` dict ပျောက်ပြီး extend reminder မှာ "Plan: 1 min" ပြနေတယ်
- **Root Cause:** `_SESSION_TOTAL_MINS` က in-memory dict — JSON file ထဲမှာ persist မလုပ်ထားဘူး
- **Fix:** 
  - **session_reminder_store.py:** `persist_reminder()` မှာ `total_plan_mins` param အသစ်ထည့် → JSON ထဲမှာသိမ်း
  - **session_reminder_store.py:** `restore_reminders_async()` မှာ bot restart ချပြီးရင် `_SESSION_TOTAL_MINS` ကို JSON ကနေ ပြန်ဖြည့်ပေး
  - **booking_flow.py:** `_remind_loop()` နဲ့ `_do_extend()` နှစ်ခုစလုံးက `persist_reminder()` ခေါ်တဲ့အခါ `total_plan_mins` ပါအောင်ထည့်
- **Files:** `bot/session_reminder_store.py`, `bot/booking_flow.py`
- **Lessons:** 
  - **26.** Any state that should survive bot restart must be serialized to `session_reminders.json` — `_SESSION_TOTAL_MINS` is no exception
  - **27.** In-memory dicts (`_SESSION_TOTAL_MINS`, `_REMINDER_TASKS`, etc.) are ephemeral; always pair with a persistence layer when cross-restart survival is needed
- **Status:** ✅ Deployed + verified

## 🔧 Food Cart Not Loading in Session End Voucher (18:05 UTC)

### 8. Food Note → Session End → booking_id mismatch
- **Bug:** Food note items stored in `food_cart` table never appeared in session end voucher
- **Symptom:** Food items saved in food note step, but voucher shows empty food list at session end
- **Root Cause:** `step_end_session()` in `console.py`:
  1. Gets `bk_id` from console_status `booking_id` ✅
  2. `end_booking_async(bk_id)` — changes status to 'Done' ✅
  3. **Then** performs second booking lookup via `bookings?memberId={mbr}` — filters by status ('confirmed','arrived','in_use','Active') → 'Done' excluded → `_linked_bk_id` always empty ❌
  4. `launch_session_sale()` called with `booking_id=''` → food cart never fetched ❌
- **Fix:** Use `bk_id` from console_status directly (already found before end_booking). Only fallback to member lookup if `bk_id` was empty.
- **File:** `bot/handlers/console.py` — `step_end_session()`
- **Lesson 28:** After ending a booking, never try to look it up again by status filter — it's now 'Done'. Use the ID you already had before the status change.
- **Status:** ✅ Deployed + restart OK
