### 📝 New Lessons
- **PyMySQL `%` is dangerous in SQL strings** — `LIKE 'Topup%'` causes `ValueError: not enough arguments for format string` because PyMySQL's C-based cursor sees `%` and expects a format argument. Use `CONCAT(CHAR(84),CHAR(111),CHAR(112),CHAR(117),CHAR(112),CHAR(37))` for `Topup%` or filter in Python.
- **sed + Python strings = disaster** — sed can't properly handle nested quotes in Python strings. Use Python string replacement with `src.replace()` for safe modification.
- **JavaScript `\x52\x65\x70\x6c\x61\x63\x65` trick**: When shell heredocs break on `()`, use encoded strings in the sed command.
- **GROUP BY + pipe-delimited fields = wrong aggregation** — When topup records store multi-payment info as `KPay:90000/Cash:0`, `GROUP BY payment_method` collapses all records sharing the same `payment_method` column value and loses individual segment parsing.
