### Key Lessons
1. **DB account naming mismatch**: `cash_movements` table stores account as "KBZ Bank", "ACM's Acc" (human labels) but code uses keys ("kbz_bank", "acm_acc"). Always maintain a mapping dict.
2. **`transfer_out` sign convention**: Amounts stored as NEGATIVE in `cash_movements` → use `+ t_out` (not `- t_out`).
3. **Readable Balance Sheet**: Final endpoint reuses exact same income-parsing logic as `finance/balances` to guarantee consistency.
4. **Router/index.ts fragility**: When adding multiple routes, ensure all closing braces/commas are preserved. One missing `},` breaks entire Vite build.

### What Was Built

#### A. Asset Register Enhancements

#### B. Backend Changes (`dashboard_routes.py`)
- **POST /assets/create**: Auto-calcs monthly_dep, months_elapsed, acc_depreciation, book_value
- **PUT /assets/{id}/dispose**: Accepts `return_account` param (default "Cash"), injects proceeds
- **Asset total**: `WHERE status='active'` filter — disposed assets excluded
- **Balance Sheet**: Gross→Dep→NBV structure; Depreciation Reserve in Equity; Balanced A=L+E
- **P&L**: Depreciation expense (non-cash) separated; Operating Profit before Dep added
- **Cash Flow**: Depreciation add-back; Net Cash Flow before Financing added

#### C. Depreciation Formula (Straight-Line)
```
monthly_dep = max(0, (amount - salvage) / (useful_life * 12))
months_elapsed = (June 1, 2026 - purchase_date) in months
acc_depreciation = min(monthly_dep * months_elapsed, amount - salvage)
book_value = amount - acc_depreciation
```
Base date: **June 1, 2026** (static, not datetime.now())

#### D. Report Enhancements
| Report | New Lines |
|--------|-----------|
| **P&L** | Operating Profit (before Dep), Depreciation (non-cash), Net Profit (after Dep) |
| **Balance Sheet** | Gross Fixed Assets, Accumulated Depreciation, NBV, Depreciation Reserve in Equity |
| **Cash Flow** | Net Cash before/after Dep Add-back, Net Cash Flow before Financing |
