
## 🐛 Expired Display Fix — VPN Admin UI

### Bug
Osmo reported: Expired keys showing **"-5d ago"** instead of "Expired 5d ago" in the admin/agent UI.

### Root Cause
`Test_Trial_0001` was auto-expired by data-limit checker (`is_active=0` via `mark_expired()`), but `expires_at` field was left unchanged at `2026-07-14` (future). In `get_key_status()`, `(now - exp).days` returns a **negative number** when `exp` is in the future.

### Fix
`/opt/outline-web/server.py` — `get_key_status()`:
- Added `if days_ago >= 0`: show "Expired Xd ago" (normal case)
- `else`: show "🔴 Revoked" instead of "Expired -Xd ago"
