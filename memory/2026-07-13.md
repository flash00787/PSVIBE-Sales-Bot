# Memory (2026-07-13) — Deposit Amount Fix + Food Menu Markdown Fix

## Deposit Amount Display Fix 🐛→✅
- **Problem:** Dashboard timeline showed deposit_amount=0 for ALL bookings despite DB having correct values. SQL SELECT in `api_get_bookings()` was missing `cb.deposit_amount`.
- **Root Cause:** When adding `deposit_amount` to the normalized response dict, we forgot to add `cb.deposit_amount` to the SQL SELECT statement. Deposit_status was in SELECT but deposit_amount wasn't.
- **Fix:** Added `cb.deposit_amount` to SQL SELECT at line 671 of `booking_routes.py`.
- **Lesson:** Always verify that new response fields are also included in the SQL SELECT statement.

## Food Menu Markdown Parsing Error 🐛→✅
- **Problem:** Customer Bot `/foodmenu` command crashed with "Can't parse entities: can't find end of the entity starting at byte offset 1781"
- **Root Cause:** Inventory item names with underscores (`_`) — e.g., `လျှာဂျွမ်းထိုး_ဟော့ဟော့`, `Spicy_Sasuge_Stick` — Telegram Markdown interprets `_` as italic marker but can't find closing `_`.
- **Fix:** Added `safe_name = name.replace('_', '\\_').replace('*', '\\*').replace('`', '\\`').replace('[', '\\[')` in `cmd_food_menu()` at `customer_bot/handlers.py:475`.
- **Data:** Affected items: `လျှာဂျွမ်းထိုး_ဟော့ဟော့`, `Spicy_Sasuge_Stick`, `Yum Top_Sasuge` (underscore in name)
