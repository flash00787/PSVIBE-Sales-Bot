## DEPOSIT DEDUCTION ROOT CAUSE (Post-19:44 UTC)

### Flow Analysis — Booking #1652 (Session #1384, C-01)

**Complete call chain traced back-to-back:**
1. `step_end_confirm` (console.py:674) → `launch_session_sale(booking_id=1652)` ✅ (bk_id forwarded)
2. `step_sale_confirm` (sales.py:1490-1518) → confirms food, shows review
3. User clicks "Confirm & Save" → `step_sale_confirm` runs again 
4. Receipt sent (19:44:14) → `asyncio.create_task(_sale_bg())` at line 1637 ✅
5. ⚠️ **API restart at 19:44:24** (garbled Unicode fix applied)
6. `_sale_bg` runs AFTER restart → **API returns 401 Unauthorized** 
7. `_do()` sets `_sales_api_ok = False` → **silently skips deposit redemption** ❌
8. **No log output** — code at lines 1553-1554 had NO logging for non-success API responses

### Root Cause
`s_ale_bg()` → `_do()` → `api_add_sales_record()` returns error. Code has `try/except` for exceptions but **no logging for non-200 API responses**. When API returns 401 (or any non-success), `_result.get("success")` is `False`, `_sales_api_ok = False`, and the code continues SILENTLY. No warning, no error log.

### Fix Applied
- **File:** `/root/psvibe-sales-bot/bot/handlers/sales.py:1553-1558`
- Added: `logging.warning("Sales API returned error: %s", json.dumps(_result, ensure_ascii=False, default=str))` when `_sales_api_ok` is False
- **Service restarted:** `psvibe-sale-bot` (19:58 UTC)

### New Lesson (#163)
| # | Lesson |
|:-:|--------|
| 163 | **API background tasks must log non-success responses** — `_sale_bg._do()` silently swallows API errors (401/500). Always log non-200 responses explicitly. Silent failure → undetected data loss (deposit not redeemed, sales not recorded). |
