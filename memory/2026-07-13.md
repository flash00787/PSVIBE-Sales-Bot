## 📌 Fix: booking_id NOT passed to launch_session_sale (Jul 13, 18:20 UTC)

### The Bug
`_end_single_session_and_launch` (sales.py:674) and `step_ds_console_in_session` (sales.py:826) extracted `bk_id` from the active session but **did not forward it** to `launch_session_sale`. The booking_id defaulted to `""`, so:

- `launch_session_sale` never fetched deposit info
- `step_sale_confirm` never displayed deposit line
- API never received `booking_id` → deposit deduction SKIPPED

### The Fix
Added `booking_id=bk_id` to both call sites:
- `sales.py:707` → `launch_session_sale(..., booking_id=bk_id)`
- `sales.py:850` → `launch_session_sale(..., booking_id=bk_id)`

### Verification
- `console.py:581` (`step_end_confirm`) already correctly passed `booking_id=_linked_bk_id` ✅
- Combined bill path (`sales.py:771`) still doesn't pass booking_id — rare multi-console case, separate issue
