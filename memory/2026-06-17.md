## 🎯 CRITICAL BUG FIX — Bookings Losing Names on Approve (07:00+ UTC)

### Bug Report
Boss reported: Confirmed bookings in Sales Bot don't show customer names. Admin Group notification shows names (like "Yoon", "Naing Htet Aung") but Sales Bot's `/confirm` shows only 📞 phone numbers.

### Root Cause Found — `COALESCE('', name)` Trap
**Location:** `app.py` line 1335 — `PATCH /api/bookings/{id}/status` endpoint

```python
# BEFORE (BUG)
"staff_name=COALESCE(%s, staff_name), notes=COALESCE(%s, notes)"
# %s = req.get("staffName", "") → ""
# COALESCE('', 'Naing Htet Aung') → '' ← RETURNS EMPTY STRING!
# Because '' IS NOT NULL in SQL!
```

**Flow:**
1. Customer bot creates booking → POST with `customerName: "Naing Htet Aung"` → INSERT stores `staff_name = "Naing Htet Aung"` ✅
2. Admin notification sent with correct name ✅
3. Staff clicks "Approve" → PATCH with `{"status":"confirmed","staffNote":"Approved by Osmo"}` — NO `staffName` field
4. `req.get("staffName", "")` → `""` (empty string)
5. `COALESCE('', 'Naing Htet Aung')` → `''` 💀 **NAME WIPED!**
6. Same for `notes`: `COALESCE('', 'Game: FC 26 | Phone: ... | User: ...')` → `'Approved by Osmo'` 💀

### The Fix
```python
# AFTER (FIXED)
"staff_name=COALESCE(NULLIF(%s, ''), staff_name), notes=COALESCE(NULLIF(%s, ''), notes)"
# NULLIF('', '') → NULL
# COALESCE(NULL, 'Naing Htet Aung') → 'Naing Htet Aung' ✅
```

### Critical Lesson
**`COALESCE('', val)` returns `''` because empty string IS NOT NULL in SQL.** When using COALESCE as "use this new value, or keep existing if not provided", always wrap the new value with `NULLIF(%, '')` to treat empty string as "not provided."
