# 2026-06-17

## Suggestion System — Full Integration (Discord + Web Dashboard)

### What Was Done

#### 1. Discord Bot — Staff Review Commands (`/root/psvibe-discord-bot/bot.js`)
Converted the `/suggest` command from a simple input to subcommand-based:
- `/suggest create <game> [reason]` — Submit a suggestion (same as before, now a subcommand)
- `/suggest approve <id>` — **Staff only** — Sets status to "approved", updates embed in #suggestions with green ✅
- `/suggest reject <id> [reason]` — **Staff only** — Sets status to "rejected", updates embed with red ❌
- `/suggest list` — **Staff only** — Shows all pending suggestions
- `/suggest view <id>` — Anyone — View any suggestion by ID

On approve/reject, the bot fetches the original message in #suggestions and edits the embed (changes color, title, fields).

#### 2. deploy-commands.js
Updated to register all 5 subcommands under `/suggest`. Deployed successfully (35 commands registered).

#### 3. API Server (`/root/psvibe_api_server/app.py`)
Added 3 new endpoints:
- `GET /api/suggestions?status=pending|approved|rejected` — List suggestions with optional filter
- `GET /api/suggestions/<id>` — Get a single suggestion
- `PUT /api/suggestions/<id>` — Update status (body: `{ status: "approved"|"rejected", reviewedBy: "name", reason: "optional" }`)

Reads/writes from `/root/psvibe-discord-bot/suggestions.json` (same file as Discord bot).

#### 4. Kora Dashboard (`/root/.openclaw/workspace/kora_dashboard/index.html`)
Added "💡 Suggestions" tab with:
- Stats cards (Total, Pending, Approved, Rejected)
- Filter dropdown (All/Pending/Approved/Rejected)
- Sortable table with ID, Suggestion, User, Status, Votes, Date
- Action buttons: View 👁️, Approve ✅, Reject ❌ (for pending suggestions)
- Detail modal with full suggestion info
- English + Myanmar translations

#### 5. Data Normalization
- Fixed suggestion #1 which was missing `status` field (old format)
- API now treats missing status as 'pending'
