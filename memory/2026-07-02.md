## Evening Session — Kora Memory System v2.0 Discussion & Gaps (18:49 UTC)

### Boss Review: "Before vs After MongoDB"
Boss asked how the memory system changed after MongoDB migration. Presented detailed comparison:

| Dimension | Before (File-only) | After (MongoDB) |
|-----------|-------------------|-----------------|
| Storage | 3,600 KB flat .md | 4,188 KB (MongoDB + thin .md) |
| Search | OpenClaw vector only | MongoDB text index + OpenClaw vector |
| Relationships | No auto-links | 310 auto-detected bug→fix→lesson chains |
| Code Knowledge | 0 entities | 7,820 entities, 11,046 relationships |
| Intelligence | Manual classification | Auto-type, severity, tags, quality scores (0-9) |
| Trace "function→endpoint→DB" | ❌ Impossible | ✅ < 0.2 sec |

### Kora's Honest Assessment — 3 Critical Gaps

**Gap #1 (🔴 Critical): Kora Doesn't Auto-Query MongoDB**
- Kora only uses `memory_search` (file-based) — never auto-queries MongoDB code graph
- Boss says "get_finance_balances bug" → Kora does slow grep/files instead of instant MongoDB trace
- Fix: AGENTS.md reflex rules so Kora auto-calls `kora_memory.py trace <name>` for function/endpoint/table names

**Gap #2 (🟡 Medium): No Auto-Refresh on Code Graph**
- Code changes daily but graph only updates on manual rebuild
- New functions/endpoints not discoverable until rebuild
- Fix: Cron jobs — incremental every 2 hours + full rebuild weekly

**Gap #3 (🟡 Medium): Fragmented CLI**
- 4 separate scripts with different command patterns
- Fix: Unified under single `kora_memory.py` with 10 subcommands

### All 3 Gaps Fixed ✅

**Gap #1 Fix — AGENTS.md MongoDB Reflex Rules**
Added "🧠 MongoDB Memory System (v2.0 — 2026-07-02)" section to AGENTS.md:
- Auto-trace on function/endpoint/table mention
- Auto-impact on file changes
- Auto-search for historical bug context
- Commands: `kora_memory.py trace|impact|search|query|code-stats|enhance|dedup|quality`

```
*/120 * * * *  kora_code_graph_refresh.py    (2-hr incremental)
```
