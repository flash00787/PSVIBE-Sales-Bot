### 9. Feedback Time Double Conversion (7:14 PM → 12:44 PM) ✅
**Bug:** Feedback created_at က `7:14 PM` ပြနေတယ် — တကယ့်အချိန်က `12:44 PM` MMT ဖြစ်သင့်။
**Root Cause:** **Double Conversion** — 
- INSERT မှာ `DATE_ADD(NOW(), INTERVAL 390 MINUTE)` → DB ထဲ MMT သိမ်း (12:44)
- Dashboard SELECT မှာ `DATE_ADD(cf.created_at, INTERVAL 390 MINUTE)` → နောက်ထပ် +6:30 (19:14)
- API response မှာ `timedelta(hours=6, minutes=30)` → နောက်ထပ် +6:30
- DB ထဲ MMT သိမ်းထားပြီးသားကို SELECT & API က UTC→MMT ထပ်ပြောင်းနေ → Double = 13 hrs ahead!

**Fix:**
1. Old entries (#48-51) — UTC→MMT conversion in DB (`UPDATE customer_feedback SET created_at = DATE_ADD(created_at, INTERVAL 390 MINUTE) WHERE id < 52`)
2. `dashboard_routes.py` — `DATE_ADD(cf.created_at, INTERVAL 390 MINUTE)` ၅ နေရာ ဖြုတ် (WHERE, SELECT, GROUP BY)
3. API response `timedelta(hours=6, minutes=30)` ဖြုတ်
**Files:** `/root/psvibe_api_server/dashboard_routes.py`, DB migration

### Lesson Learned — ⚠️ Timezone Fix Pattern
- **Single source of truth:** Either store UTC + convert on read, OR store MMT + never convert. Never mix.
- When fixing timezone: **check ALL three layers** — INSERT, SELECT, API response
- **Old data migration:** Always migrate old rows BEFORE removing conversions from SELECT queries
- Lesson #60 extended: Not just INSERTs need MMT, but also verify no double-conversion in read paths
