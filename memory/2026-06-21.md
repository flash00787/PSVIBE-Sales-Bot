## 16:00-16:50 MMT — Timeline Cancelled Booking Display

### Feature: Auto-cancelled bookings visible on timeline (2hr after cancel)
- `timeline.ts` — added `cancelled` status fetch + client-side 2hr filter via `created_at`
- `app.py` — SQL SELECT added `created_at` column
- `TimelineView.vue` — cancelled bookings rendered with gray strikethrough + cancelled badge

### 🐛 Two Bugs Found:

**Bug 1 (16:20): `created_at` missing from API response**
- SQL SELECT ထဲ `created_at` ထည့်ထားပေမဲ့ API response dict (`normalized`) ထဲ မထည့်မိ
- Frontend never receives `created_at` → 2hr filter silently discards ALL cancelled bookings
- Fix: Added `"created_at": _created_str` to normalized dict in `api_get_bookings()`

**Bug 2 (16:47): Timezone Mismatch (ROOT CAUSE)**
- MySQL TIMESTAMP stored in UTC (`2026-06-21 16:50:49`)
- `datetime.isoformat()` on naive datetime → `"2026-06-21T16:50:49"` (no timezone)
- JS `new Date()` parses timezone-less ISO string **ambiguously** (UTC on Chrome, local on Safari)
- 6.5hr MMT offset + ambiguous parsing → `created_at > twoHoursAgo` always false
- **Fix:** `_created.replace(tzinfo=timezone.utc).isoformat()` → `"2026-06-21T16:50:49+00:00"` (explicit UTC)
- JS consistently parses `"+00:00"` as UTC → correct comparison
