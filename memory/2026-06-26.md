### Fixes Applied

#### 1. Food Orders Timezone (Dashboard Routes)
- `dashboard_routes.py` food-orders endpoint: Added `CONVERT_TZ(created_at, '+00:00', '+06:30')` for `created_at`, `fulfilled_at`, `start_time` → MMT

#### 2. Sale Daily Time Column
- **API:** Added `CONVERT_TZ(created_at, '+00:00', '+06:30') as sale_time` to query, `"sale_time": str(r["sale_time"])[:16]` in response
- **Frontend source (`SaleDaily.vue`):** Added "Time" column header + body cell between Date and Console columns

#### 3. Food Orders Sorting Fix
**Root Cause:** JS object integer key sorting — `Object.values(groups).reverse()` sorted booking_ids as integers (848 < 935 < 976 < ... < 1005), so .reverse() put lower numbers at the BOTTOM. With 43 groups, BK#848 was at absolute position 43.

**Fix in `FoodOrders.vue` source:**
```ts
// Sort by Active first, then by recency (max food_cart id)
const sorted = Object.values(groups).sort((a, b) => {
  if (a.status === 'Active' && b.status !== 'Active') return -1;
  if (a.status !== 'Active' && b.status === 'Active') return 1;
  return (b.maxId || 0) - (a.maxId || 0);
});
```

#### 4. Timeline Lane-Based Stacking
**Problem:** All bookings on same console positioned at `top:5px; bottom:5px` — overlapping blocks hard to tap on mobile.

**Fix in `TimelineView.vue` source:**
- Added `computeStackLanes()` function: greedy lane assignment algorithm — each booking assigned to first lane where it doesn't overlap previous
- Added `laneMap` computed: `Map<bookingId, {lane, maxLanes}>` for all consoles
- Modified `blockStyle()`: when maxLanes > 1, sets `top: 5 + lane * rowPx`, `height: rowPx - 2`
- Row height split: 2 bookings → 21px each, 3 bookings → 14px each
- Mobile touch targets: `min-width: 32px`, `min-height: 18px`, `hover:z-30`

#### 5. Timeline Rebuild Recovery
- Edit minified `TimelineView-DsSlmMsS.js` directly → broke due to `et` name conflict (import alias `d as et` overwritten)
- Recovered by finding source project: `/root/psvibe-dashboard/`
- Source `TimelineView.vue` found and edited properly
- Build: `cd /root/psvibe-dashboard && npx vite build`
- Three build cycles: first successful, second after FoodOrders+SaleDaily edits, third after CSS file accidentally deleted
- Old minified JS files deleted; new ones with different hashes generated

### Key Lessons Learned

**#28: Never edit minified Vue build output directly** — Always edit the source `.vue` files and rebuild. Single-character name conflicts can break the entire component. The project is at `/root/psvibe-dashboard/`.

**#29: Vite build replaces all hashes** — Every rebuild generates new content hash filenames. Old files must be cleaned up to avoid serving stale cached versions.

**#30: JS object key sorting trap** — String keys that look like integers (e.g., "848") are sorted numerically by V8, not by insertion order. Using `.reverse()` on `Object.values()` doesn't reverse insertion order when keys are numeric strings.

## Session: Timeline Stacking v2 → Popup Approach (~11:00-12:46 UTC)

### Timeline Lane Stacking v2 — REJECTED by Boss

**Problem with v1:** Stacked blocks too small to read (14px each for 3 overlapping bookings)

**v2 Attempt:** Dynamic row height — `min-h-[42px]` + `getRowHeight()` computed:
- 1 lane: 42px, 2 lanes: 44px (22px each), 3 lanes: 66px (22px each)
- Each lane gets at least 22px
- Changed `h-[42px]` to dynamic `:style="{ height: getRowHeight(c) + 'px' }"`

**Boss Feedback:** "ကြည့်ရတာ အဆင်မပြေဘူး" — lane-based stacking fundamentally doesn't look good

**Boss's Alternative:** နဂိုအတိုင်းထား + stack ဖြစ်နေတဲ့နေရာ နှိပ်ရင် popup ပြ

### Timeline — Final Version: Popup Approach

#### 1. Reverted to Original Layout
- `git restore` `TimelineView.vue` — removed all lane stacking code
- Removed: `computeStackLanes()`, `laneMap`, `getRowHeight()`, `MIN_LANE_H`
- Restored: `h-[42px]` fixed row, `top-[5px] bottom-[5px]` block positioning
- Blocks overlap naturally as before

#### 2. Kept Mobile Touch Targets
- `min-w-[6px]` → `min-w-[32px]` — minimum touch width
- Added `min-h-[18px]` — minimum touch height
- `hover:z-20` → `hover:z-30` — higher stacking on hover/tap
- These stay in the restored template

#### 3. Added Stack Popup Feature
**Logic (`handleBookingClick`):**
1. Get all bookings for this console
2. Find ones whose time range overlaps the clicked booking
3. If only 1 (self-only) → open detail modal directly
4. If 2+ overlapping → show popup listing all overlapping bookings

**Popup template:**
```html
<Teleport to="body">
  <div v-if="stackPopup.visible" class="fixed inset-0 z-[100]" @click.self="closeStackPopup">
    <div class="absolute bg-white dark:bg-gray-800 rounded-lg shadow-2xl border border-gray-200 dark:border-gray-700 p-3 min-w-[240px] max-w-[300px] max-h-[70vh] overflow-y-auto">
      <div class="text-xs font-bold text-gray-500 dark:text-gray-400 mb-2">
        📋 {{ stackPopup.bookings.length }} overlapping bookings
      </div>
      <div v-for="b in stackPopup.bookings" @click="openStackedBooking(b)" 
           class="flex items-center gap-2 px-2 py-1.5 rounded cursor-pointer hover:bg-gray-100 dark:hover:bg-gray-700 text-sm">
        <span class="w-2 h-2 rounded-full flex-shrink-0" :class="statusDotClass(b.status)"></span>
        <span class="font-medium truncate">{{ b.customerName || b.phone || '—' }}</span>
        <span class="text-gray-400 text-xs ml-auto whitespace-nowrap">{{ fmtTime(b.startTime) }}–{{ fmtTime(b.endTime) }}</span>
      </div>
    </div>
  </div>
</Teleport>
```

**`openStackedBooking` handler:**
- Closes popup
- Opens modal with selected booking data

#### 4. Fixed Template `window` Issue
- `innerWidth`/`innerHeight` not accessible in Vue template → bound popup position via `style` injection in `handleBookingClick` using `popupStyle` computed
