#### 3. Food Orders Sorting Fix
**Root Cause:** JS object integer key sorting — `Object.values(groups).reverse()` sorted booking_ids as integers (848 < 935 < 976 < ... < 1005), so .reverse() put lower numbers at the BOTTOM. With 43 groups, BK#848 was at absolute position 43.

**Fix in `FoodOrders.vue` source:**
```ts
// Sort by Active first, then by recency (max food_cart id)
const sorted = Object.values(groups).sort((a, b) => {
  if (a.status === 'Active' && b.status !== 'Active') return -1;
  if (a.status !== 'Active' && b.status === 'Active') return 1;
  return (b.maxId || 0) - (a.maxId || 0);
});
```

### Key Lessons Learned

**#28: Never edit minified Vue build output directly** — Always edit the source `.vue` files and rebuild. Single-character name conflicts can break the entire component. The project is at `/root/psvibe-dashboard/`.

**#29: Vite build replaces all hashes** — Every rebuild generates new content hash filenames. Old files must be cleaned up to avoid serving stale cached versions.

**#30: JS object key sorting trap** — String keys that look like integers (e.g., "848") are sorted numerically by V8, not by insertion order. Using `.reverse()` on `Object.values()` doesn't reverse insertion order when keys are numeric strings.
