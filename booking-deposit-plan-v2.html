📌 Section 1 — Payment Method Tracking

Problem

Customer က WavePay နဲ့ deposit ပေးတယ်။ ဒီငွေက WavePay account ထဲကိုဝင်တယ်။ ဒါပေမယ့် BS မှာ account balance တွေကိုခွဲမှတ်တဲ့အခါ "deposit method အတိုင်း မှတ်ထားမှ BS ကိုက်မယ်"

Solution: deposit_method + cash_movements

DB bookings.deposit_method + cash_movements နဲ့ အောက်ပါအတိုင်း သေချာခွဲမှတ်မယ်:

Financial Recording (Settlement Moment)

DB Settlement button နှိပ်တာနဲ့ တစ်ပြိုင်နက် —

-- 1. Record full sale in sales_daily
--    PM shows original deposit method + remaining payment
INSERT INTO sales_daily (total, payment_method, ...)
VALUES (10000, 'WavePay:7500(DepositRedeemed)|Cash:2500', ...);

-- 2. Record remaining payment as cash_movements inject
--    (deposit portion ALREADY in WavePay — no new inject needed)
INSERT INTO cash_movements (account, movement_type, amount, note)
VALUES ('Cash', 'inject', 2500, 'Game revenue #212 (remainder)');

-- 3. Clear deposit liability
--    Eject from deposit_pending tracking
INSERT INTO cash_movements (account, movement_type, amount, note, category)
VALUES ('WavePay', 'deposit_eject', 7500, 'Booking #212 deposit redeemed', 'deposit_redemption');

-- 4. Update booking status
UPDATE bookings SET deposit_status = 'redeemed', status = 'done' WHERE id = 212;

📌 Section 4 — Balance Sheet Update

API finance_routes.py BS function ကို deposit အတွက် update လုပ်မယ်:

# --- NEW: Deposit Liability ---
# Fetch all bookings with verified deposits not yet redeemed
deposit_liability = db.query("""
    SELECT COALESCE(SUM(deposit_amount), 0)
    FROM bookings
    WHERE deposit_status IN ('verified', 'paid')
      AND status IN ('confirmed', 'checked_in')
""").fetchone()[0]

# --- Modify Retained Earnings ---
# Already-verified deposits should NOT count as revenue yet
# Only deposits that were redeemed (played) count as revenue
deposit_played = db.query("""
    SELECT COALESCE(SUM(deposit_amount), 0)
    FROM bookings
    WHERE deposit_status = 'redeemed'
      AND MONTH(created_at) = :m AND YEAR(created_at) = :y
""").fetchone()[0]

# Revenue from sales_daily already includes deposit portion
# via payment_method parsing
retained_earnings = (sales_total - opex - cogs - dep)
                 + deposit_played  # forfeited deposits
                 - deposit_outstanding  # unplayed deposits (liability)
⚠️ This is critical for BS accuracy!
ဒီ logic မပါရင် လက်ရှိ 184K gap လိုမျိုး ထပ်ဖြစ်လိမ့်မယ်။ Deposit ကို liability အနေနဲ့ သီးသန့်ခွဲမှတ်မှ BS ကကိုက်မယ်။
