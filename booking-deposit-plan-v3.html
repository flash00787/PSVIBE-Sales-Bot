📌 3. Redemption Flow (ဆိုင်ရောက်လို့ ပြန်နှုတ်)

Scenario: Booking #212 — PS5 2hr = 10,000 Ks, Deposit 7,500 Ks (KPay)

Step-by-Step

#Stepdeposit_statusBookingFinancial
1 Customer books PS5 2hr → deposit 7,500 Ks (KPay) pending pending —
2 Customer transfer 7,500 Ks via KPay → sends ref no paid pending cash_movements: inject (KPay, +7,500, deposit_pending)
KPay balance ↑7,500 — BS liability ↑7,500
3 Staff verifies deposit ✅ verified confirmed Still liability (not yet revenue)
4 Customer arrives → checks in → plays PS5 2hr verified checked_in Timer starts normally
5 Session ends → Staff settles redeemed done 3 things happen at once 👇

Behind the scenes (one click)

-- ① Record full sale in sales_daily
-- PM shows: original deposit method (KPay) + remaining payment method
INSERT INTO sales_daily (total, payment_method, ...)
VALUES (10000, 'KPay:7500(DepositRedeemed)|KPay:2500', ...);

-- ② Record remaining payment into cash_movements
--    (deposit 7,500 already in KPay — NO new inject needed)
INSERT INTO cash_movements (account, movement_type, amount, note)
VALUES ('KPay', 'inject', 2500, 'Game revenue #212 (remainder)');

-- ③ Clear deposit liability
--    deposit_redemption eject = mark liability as settled
UPDATE bookings SET
  deposit_status = 'redeemed',
  deposit_redeemed_at = NOW(),
  status = 'done'
WHERE id = 212;

📌 4. Edge Cases

4A — No-Show (Booking timeout without check-in)

deposit_status → "forfeited"

Financial: 7,500 Ks → No-Show Revenue
cash_movements: no new movement (money already in KPay)
BS: liability (-7,500) + revenue (+7,500) ✅
PNL: separate line "No-Show Deposit Revenue"

Customer notified: "သင်၏ deposit 7,500 Ks ကို no-show အတွက်
သိမ်းလိုက်ပါသည်"

4B — Shop Cancels (booking cancelled by staff after deposit verified)

deposit_status → "refunded"

Financial: 7,500 Ks → Refund to customer
cash_movements: eject (KPay, -7,500, deposit_refund)
BS: KPay balance ↓7,500, liability ↓7,500 ✅

Customer notified: "စတိုးဆိုင်ကိစ္စကြောင့် Booking ကို
ပယ်ဖျက်ပါသည် — သင်၏ deposit 7,500 Ks ကိုပြန်အမ်းပါမည်"

4C — Customer Cancels Before Paying

deposit_status → "none" (was still "pending")
No financial impact. Just cancel booking.

4D — Customer Pays Less Than Deposit

Keep checking: deposit_amount = calculated amount (non-negotiable)
If customer sends wrong amount → Staff rejects → "ငွေလွှဲပမာဏမှားနေပါသည်
— 7,500 Ks အတိအကျလွှဲပေးပါ"

4E — Deposit Method ≠ Session Payment Method

Example: Deposit via KPay (7,500 Ks), remaining paid via Cash (2,500 Ks)
sales_daily PM: "KPay:7500(DepositRedeemed)|Cash:2500"
→ BS can trace deposit method AND remaining payment method separately
→ No confusion

📌 5. Balance Sheet Update

API finance_routes.py BS function မှာ —

# ── NEW: Deposit Liability ──
# Bookings with deposit_status = 'verified' or 'paid' = still liability
deposit_liability = db.query("""
    SELECT COALESCE(SUM(deposit_amount), 0)
    FROM bookings
    WHERE deposit_status IN ('verified', 'paid')
      AND status IN ('confirmed', 'checked_in')
""").fetchone()[0]

# ── Modify Equity Calculation ──
# Revenue from deposits redeemed this month
deposit_redeemed = db.query("""
    SELECT COALESCE(SUM(deposit_amount), 0)
    FROM bookings
    WHERE deposit_status = 'redeemed'
      AND deposit_redeemed_at IS NOT NULL
      AND MONTH(deposit_redeemed_at) = :m
      AND YEAR(deposit_redeemed_at) = :y
""").fetchone()[0]

# Revenue from forfeited deposits
deposit_forfeited = db.query("""
    SELECT COALESCE(SUM(deposit_amount), 0)
    FROM bookings
    WHERE deposit_status = 'forfeited'
      AND MONTH(updated_at) = :m AND YEAR(updated_at) = :y
""").fetchone()[0]

# Adjusted retained earnings
retained = sales_total - opex - cogs - dep
         + deposit_forfeited    # forfeited = income
         - deposit_outstanding  # not yet played = liability

# ── NEW: Add deposit_liability to Liabilities side
liabilities = member_liability + deposit_liability + ...
