    Error: assert '2' in '🎮 <b>Game Library</b> (0 ဂိမ်း)...'
           Game count shows 0 even though 2 games were mocked.

    Root cause: The test patches `games.fetch_games` but `show_game_menu` 
    imports fetch_games from bot at module level:
      from bot import (..., fetch_games, fetch_games_async, ...)
    So `games.fetch_games` is a local reference bound at import time.
    Patching the module-level attribute AFTER it's been imported into the
    local namespace doesn't affect the already-bound variable.
    The real `bot.fetch_games` (still unmocked) returns [] → count is 0.
    
    Fix: Instead of `games.fetch_games = MagicMock(...)`, do:
      `sys.modules['bot'].fetch_games = MagicMock(return_value=[...])`
      THEN import games (or use patch context manager)
    Fix effort: Medium (5-10 min — needs import order fix)
