  ADD COLUMN deposit_status      ENUM('none','pending','paid','verified','refunded','forfeited')
                                                            DEFAULT 'none',
  ADD COLUMN deposit_method      VARCHAR(32)     DEFAULT NULL    COMMENT 'kpay|wavepay|bank_transfer|wallet',
  ADD COLUMN deposit_ref         VARCHAR(128)    DEFAULT NULL    COMMENT 'Customer reference no / screenshot file path',

  ADD COLUMN deposit_verified_at DATETIME        DEFAULT NULL;
💡 deposit_status states:
none → (deposit not required for this booking / legacy)
pending → (waiting for customer to pay & upload proof)
paid → (customer uploaded proof, waiting for staff verify)
verified → (staff confirmed deposit received → booking confirmed)
refunded → (booking cancelled by shop after verified — deposit returned)
forfeited → (no-show / booking cancelled after deposit — deposit kept)

1C — Deposit Schedule Logic

API /api/deposit/schedule မှာ အောက်ပါအတိုင်း တွက်မယ်:

def calc_deposit(console_type: str, duration_mins: int, day_of_week: int, hour: int) -> int:
    """Calculate deposit amount. Returns Ks."""
    base = 5000  # base deposit: 5000 Ks

    # Duration multiplier
    if duration_mins <= 60:       mult = 1.0
    elif duration_mins <= 120:    mult = 1.5    # 7,500 Ks
    elif duration_mins <= 180:    mult = 2.0    # 10,000 Ks
    else:                         mult = 2.5    # 12,500 Ks

    # Premium console
    if console_type.lower() == "ps5 pro":   mult *= 1.3

    # Peak hours (Fri-Sun 18:00-21:00) = +50%
    if day_of_week >= 5 and 18 <= hour < 21:  mult *= 1.5

    # Cap: min 3000, max 15000
    amount = max(3000, min(15000, int(base * mult)))
    return amount

⚠️ Financial Handling:
  • Deposit received → cash_movements (inject, deposit_pending) — liability
  • Deposit verified → booking confirmed, but still liability until session
  • Session completed → deposit converts → cash_movements eject (deposit) + sales_daily
  • No-show → cash_movements (inject, deposit_forfeit) → PNL income
